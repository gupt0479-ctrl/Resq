## User stories

- As a judge, I want to understand the survival story in under one minute.
- As a teammate, I want the rescue queue, workflow page, and integrations page
  to reinforce the same product story.
- As a frontend collaborator, I want stable TinyFish response fields and honest
  live/mock/degraded messaging.

## Acceptance criteria

- Dashboard, rescue, workflow, and integrations surfaces tell one rescue story
- Integrations page explains TinyFish status clearly
- Workflow page shows an auditable survival-oriented run history
- Setup docs match the actual repo state
